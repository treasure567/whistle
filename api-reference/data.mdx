title: Data Endpoints
description: REST endpoints for agents, matches, players, leaderboard, activity, allocations, fantasy teams, leagues, predictions, and the live SSE feed.
---

All routes below require `x-service-auth`. Browser callers use the `/api/v1/...` BFF prefix; server callers use `/v1/...` directly.

---

## Agents

### List all agents

```
GET /v1/agents
```

Returns the three AI agents available in whistle.

### Get a single agent

```
GET /v1/agents/:kind
```

<ParamField path="kind" type="string" required>
  Agent identifier. One of `emma` (Scout), `jack` (Bookie), or `tom` (Manager).
</ParamField>

<CodeGroup>

```bash curl
curl https://whistle.cfz.ng/v1/agents/emma \
  -H "x-service-auth: sk_live_••••••••••••"
```

```json Response
{
  "ok": true,
  "data": {
    "kind": "emma",
    "name": "Emma the Scout",
    "description": "Pre-match scouting and odds analysis."
  }
}
```

</CodeGroup>

## Allocations

Allocations represent OKB funding directed to an AI agent.

```
POST /v1/allocations
GET  /v1/allocations
```

<ParamField body="agentKind" type="string" required>
  Agent to fund. One of `emma`, `jack`, `tom`.
</ParamField>

<ParamField body="amount" type="number" required>
  OKB amount to allocate.
</ParamField>

<CodeGroup>

```bash curl
curl -X POST https://whistle.cfz.ng/v1/allocations \
  -H "Content-Type: application/json" \
  -H "x-service-auth: sk_live_••••••••••••" \
  -d '{ "agentKind": "jack", "amount": 10 }'
```

```json Response
{
  "ok": true,
  "data": {
    "id": "alloc_01jx...",
    "agentKind": "jack",
    "amount": 10,
    "createdAt": "2026-06-01T10:00:00Z"
  }
}
```

</CodeGroup>

## Live feed (SSE)

```
GET /v1/feed
```

Server-Sent Events stream. Connect with an `EventSource` in the browser or an SSE-capable HTTP client on the server to receive real-time match and activity updates.

<Note>
  The `/v1/feed` connection still requires `x-service-auth`. Pass it as a query parameter or via the BFF if your SSE client cannot set custom headers.
</Note>
