## POST /v1/predictions/slip

Jack the Bookie generates a personalised bet slip — a set of picks across upcoming matches that respect your budget and risk appetite.

**Request**

<ParamField body="budget" type="number" required>
  Total OKB budget available for the slip.
</ParamField>

<ParamField body="risk" type="string">
  Risk profile. One of `"safe"`, `"balanced"` (default), or `"aggressive"`.
</ParamField>

<ParamField body="preferences" type="any">
  Optional free-form preferences (teams, markets, etc.) passed to the model as context.
</ParamField>

<ParamField body="count" type="integer">
  Number of picks to include. Defaults to a model-chosen value.
</ParamField>

**Response**

<ResponseField name="picks" type="array">
  List of pick objects:
  - `matchExternalId` (string) — identifies the match
  - `homeCode` (string)
  - `awayCode` (string)
  - `market` (string) — e.g. `"1X2"`, `"BTTS"`
  - `side` (string) — the selected outcome
  - `stake` (number) — OKB allocated to this pick
  - `confidence` (number) — model confidence 0–1
  - `note` (string) — rationale for the pick
</ResponseField>
<ResponseField name="totalStake" type="number">Sum of all pick stakes. Will not exceed `budget`.</ResponseField>
<ResponseField name="budget" type="number">The budget echoed back.</ResponseField>
<ResponseField name="source" type="string">`"llm"` or `"heuristic"`.</ResponseField>

<CodeGroup>

```bash curl

  -H "Content-Type: application/json" \
  -H "x-service-auth: sk_live_••••••••••••" \
  -d '{
    "budget": 50,
    "risk": "balanced",
    "count": 3
```

## POST /v1/fantasy/ai-pick

Emma the Scout selects a legal 15-player fantasy squad from the player pool that fits your budget and style.

**Squad legality constraints**

<Note>
  Every squad returned by this endpoint is guaranteed to satisfy all of the following rules:
  - Exactly 15 players: 2 GK, 5 DEF, 5 MID, 3 FWD
  - Exactly 11 starters; the remaining 4 are bench
  - Exactly 1 captain and 1 vice-captain among the starters
  - Total price of all 15 players does not exceed `budget`
</Note>

**Request**

<ParamField body="countries" type="array" required>
  List of country name strings to filter the player pool. Pass an empty array to use all available players.
</ParamField>

<ParamField body="strength" type="string" required>
  Squad-building strategy. One of `"balanced"`, `"galacticos"`, `"value"`, `"attacking"`, or `"defensive"`.
</ParamField>

<ParamField body="budget" type="number" required>
  Maximum total price for all 15 players.
</ParamField>

**Response**

<ResponseField name="picks" type="array">
  15-item list of player selection objects:
  - `playerId` (string) — references a player in `GET /v1/players`
  - `starter` (boolean) — `true` for the 11 starters
  - `captain` (boolean) — `true` for exactly one player
  - `viceCaptain` (boolean) — `true` for exactly one player
  - `benchOrder` (integer | null) — `1`–`4` for bench players, `null` for starters
</ResponseField>
<ResponseField name="source" type="string">`"llm"` or `"heuristic"`.</ResponseField>

## POST /v1/predictions

Record a user prediction (not AI-generated). Use `GET /v1/predictions` to retrieve stored predictions.

**Request**

<ParamField body="matchExternalId" type="string" required>
  The match this prediction applies to.
</ParamField>

<ParamField body="market" type="string" required>
  Market being predicted, e.g. `"1X2"`.
</ParamField>

<ParamField body="side" type="string" required>
  The outcome selected, e.g. `"Home"`, `"Draw"`, `"Away"`.
</ParamField>

<ParamField body="stake" type="number" required>
  OKB staked on this prediction.
</ParamField>

**Response**

Returns the created prediction object wrapped in the standard envelope.
